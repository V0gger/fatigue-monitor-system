# 👁️ Система мониторинга усталости операторов (Fatigue Monitoring System)

> Выпускная квалификационная работа по специальности "Информационные системы и технологии".

Информационная система для контроля безопасной деятельности сотрудников при выполнении технологических операций. Система использует методы компьютерного зрения для анализа состояния оператора в реальном времени (отслеживание морганий, расчет EAR и PERCLOS) и предупреждает о признаках усталости или микросна, сохраняя статистику в базу данных для последующего анализа диспетчером.

## 🛠 Стек технологий

- **Язык**: Python 3.9+
- **Компьютерное зрение**: OpenCV, dlib, imutils, SciPy
- **Backend (API)**: Flask
- **База данных**: PostgreSQL + psycopg2
- **Интерфейс**: HTML/CSS/JS (Dashboard диспетчера)
- **Безопасность**: `python-dotenv` для управления переменными окружения

## ⚙️ Установка и запуск

1. Клонирование и подготовка окружения
git clone <ссылка_на_твой_репозиторий>
cd fatigue-monitor-system
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt

2. Настройка переменных окружения
Создайте файл .env в корневой папке на основе .env.example и укажите ваши данные для подключения к PostgreSQL:
DB_HOST=localhost
DB_NAME=fatigue_monitor
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_PORT=5432

3. Инициализация базы данных
Выполните SQL-скрипт database/schema.sql в вашей СУБД PostgreSQL для создания таблиц users, sessions и events.

4. Загрузка модели dlib
Для работы детекции скачайте файл весов shape_predictor_68_face_landmarks.dat (например, с официального репозитория dlib) и поместите его в папку client_app/. (Файл исключен из Git из-за большого размера).

5. Запуск системы
Запуск клиентского приложения (детекция):
python client_app/main.py

Запуск панели диспетчера (веб-интерфейс):
python dispatcher_panel/app.py

После этого панель диспетчера будет доступна по адресу: http://127.0.0.1:5000

## 🚀 Ключевые особенности реализации

Алгоритм EAR (Eye Aspect Ratio): Расчет соотношения сторон глаза для точного определения факта моргания.
Метрика PERCLOS: Оценка процента времени закрытия глаз за заданный интервал (60 сек) для выявления накопленной усталости.
Адаптивная калибровка: При первом запуске система автоматически определяет индивидуальные анатомические особенности глаз пользователя для установки персонального порога срабатывания (EYE_AR_THRESH).
Многоуровневая система оповещений:
Визуальные рекомендации на экране.
Звуковые сигналы (winsound) при долгом моргании или микросне.
Логирование инцидентов с уровнем критичности (severity) в PostgreSQL.
Безопасность: Чувствительные данные (пароли БД) вынесены в переменные окружения, исключена жесткая привязка конфигурации к коду.
